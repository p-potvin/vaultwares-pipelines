## What the sync tool guarantees

- The content between `MANAGED:START` and `MANAGED:END` is byte-identical across every host file in every consumer repo.
- Repo-specific content (everything outside the markers) is preserved.
- Running with `--check` in CI catches drift before it merges.
- If the markers are missing, the tool appends a fresh block at the end of the file.
- If the file doesn't exist, the tool creates it.

## What the sync tool will NOT do

- Touch repo-specific content outside the markers.
- Touch the markers themselves.
- Reorder existing repo content.
- Delete a host file that's no longer in `HOST_TARGETS` (manual cleanup).
